The Science Behind the Anomaly

Why does adding more memory sometimes break things?

The History

In 1969, computer scientist László Bélady demonstrated a counter-intuitive phenomenon. While attempting to optimize memory patterns, he discovered that for certain reference strings, increasing the page frame count (memory size) resulted in more page faults, not fewer.

The Stack Property

Why does this happen? It comes down to the "Stack Inclusion Property". Algorithms like LRU and OPTIMAL are "Stack Algorithms"—meaning the pages contained in a memory of size N are always a subset of pages in memory of size N+1.

FIFO is not a Stack Algorithm. Because it blindly removes the oldest page, a slight change in frame size can completely desynchronize the "age" of pages, causing popular pages to be evicted just before they are needed.

The Classic Example
Ref String: 1, 2, 3, 4, 1, 2, 5, 1, 2, 3, 4, 5
